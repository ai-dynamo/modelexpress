## Status legend

- **In CI** ŌĆö matrix entry is uncommented, the test runs on every PR, currently passing.
- **Scaffolded, gated** ŌĆö files exist in the repo, matrix entry is commented with an explicit blocker. Easy to enable when the blocker clears.
- **Partial** ŌĆö some aspect of the area is covered as a side effect of another row, but no targeted test exists.
- **Covered implicitly** ŌĆö no separate test, but the functionality is exercised by another test that would fail if it broke.
- **Not started** ŌĆö no scaffold, no matrix entry, no dependency planning yet.

When a row moves status, update this table in the same PR so the doc stays the source of truth on what CI does and doesn't catch.
